What is Agile?

Agile is a form of project management that utilizes breaking a project into several phases. The main principals for Agile are Tasks, User Stories, Epics, and Features. Agile focuses heavily on collaboration between individuals. Often promoting collaboration between stakeholders and higher-ups.

Agile focuses on how your team should think and interact with one another, not how a team should work. The overall goal of Agile is to constantly adapt to the work you are given to find better ways to accomplish goals. Agile will often combine other methodologies to improve the Agile workflow. Take Scrum for example, here we have daily meetings, sprints, and demos. Following the Scrum method, you are not using Agile unless you are using scrum to align with Agile values and principals.


Agile Values and Principals

Agile has 4 key values. These are values are:

1. Individuals and Interactions OVER Processes and Tools
2. Working Software OVER Comprehensive Documentation
3. Customer Collaboration OVER Contract Negotiation
4. Responding to Change OVER Following a Plan

By implementing these values to projects, teams can become more reactive rather than proactive. It is important to be able to change course of action as a plan rarely is executed perfectly.

Agile also has 12 principles. These principles help support the agile mentality.

1. Priority is to satisfy the customer through early and continuous delivery of valuable software
2. Agile processes harness change for the customer's competitive advantage
3. Deliver working software frequently, with a preference
4. Business people and developers must work together daily throughout the project
5. Build projects around motivated individuals - trust them to get the job done
6. Convey information to a team with face-to-face conversation
7. Working software is the primary measure of progress
8. Agile processes promote sustainable development
9. Continuous attention to technical excellence and good design enhances agility
10. Simplicity - the art of maximizing the amount of work not done
11. The best requirements and designs emerge from self-organizing teams
12. Reflect on how to become more efficient regularly during your project

DevOps

DevOps is the combination of practices and tools to increase organization and the ability to deliver products/services at a high velocity. It can commonly be used as a form of communication between technical and non-technical teams working on the same project. During this internship you will be using Azure DevOps to track project progress, communicate with your team, and demonstrate completion of deliverables.


Git / GitHub

Git is a popular version control tool many companies and developers use on a daily basis. Note that Git and GitHub are not the same thing. Git stores your code repos while GitHub is a way to host your code and access it from anywhere. In general you will access Git through your CLI. By doing so you can clone, fork, push, and, more all without touching a GUI. That being said, you can still drag and drop code into a GitHub repository, but do so with caution as this can at times cause version control errors.

The main commands you will see using Git is pull, push, and commit. There are many other commands and configurations for each command, but here we will just scratch over the surface. Please refer to the official documentation for more details.

Pull - Incorporates changes from a remote repository into the current branch.
Commit - Creates a new commit containing the current contents of the index and log message describing the changes.
Push - Updates remote references (files) using local references (files) while sending objects necessary to complete the given references.
