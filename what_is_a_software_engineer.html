The Purpose of a Software Engineer

Let's start with a general definition of what a software engineer is. As per computerscience.org: "Software engineers design and develop computer programs and applications." Software engineers often work with system programmers, analysts, and other engineers to design systems, project capabilities, and determine performance interfaces.

The general responsibility that lie with a software engineer include:

1. Designing and maintaining software systems
2. Evaluating and testing new software programs
3. Writing and testing code
4. Produce specifications and determine operational feasibility


How Software Engineering Works

There are several flavors of software engineering. We have the application software developer who are is responsible for creating apps for iOS, Android, Windows, Linux, and other native operating systems. There are system software developers who are responsible for the hardware and software aspects of building operating systems and networks. As well as web developers (general software engineers) who create apps for the web or for multiple devices.

App focused and general software engineers are typically seen as full-stack developers while system software developers are closer to backend developers.

In most SE roles, you will work alongside a team of that can consist of data scientist, data engineers, UI/UX engineers, and many other engineering roles.

Generally software engineers need several skills in their toolkit. The goal of this internship is to prepare you as well as possible with the tools you need to assist clients when completing deliverables. Some of those tools include but are not limited to:

- HTML
- JavaScript
- Git / GitHub
- C / C++ / C#
