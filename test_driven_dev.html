What is Test Driven Development

Test Driven Development (TDD) is the process for software development practice of using unit tests cases before developing actual code. It is an iterative approach combining programming, unit test creation, and refactoring. In TDD, developers create small test cases for every feature based on their initial understanding. The primary intention of this technique is to modify or write new code only if the tests fail. This prevents duplication of test scripts.

An example would be building a calculator function. When building a calculator function, a TDD approach would involve writing a test case for the “add” function and then writing the code for the process to pass that test. Once the “add” function is working correctly, additional test cases would be written for other functions such as “subtract”, “multiply” and “divide”.

TDD differs from traditional code testing in that with TDD testing is done during development, in traditional code testing all tests are conducted after the code is complete. TDD also has a smaller testing scope because you are using unit test to test each individual function before it is deployed. In traditional code testing you are testing the entire system.


Three Phases of TDD

In TDD, you will traditionally create precises tests, correct your code, then refactor your code.

Creating Precises Tests
Developers need to create exact unit tests to verify the functionality of specific features. They must ensure that the test compiles so that it can execute. In most cases, the test is bound to fail. This is a meaningful failure as developers create compact tests based on their assumptions of how the feature will behave.

Correcting Your Code
Once a test fails, developers must make the minimal changes required to update the code to run successfully when re-executed.

Refactoring Your Code
Once the test runs successfully, check for redundancy or any possible code optimizations to enhance overall performance. Ensure that refactoring does not affect the external behavior of the program.

Benefits of TDD

TDD fosters the creation og optimized code by breaking down the codebase into small piece and ensuring correct functionality. It helps developers better analyze and understand client requirements and requests. It is easier to add and test new functionality in later stages of development again due to each individual piece being tested instead of the entire system. TDD focuses on creating tests for each functionality right from the beginning this enhances the productivity of the developer and leads to the development of a codebase that is flexible and easy to maintain.



Frameworks for TDD

Based on different programming languages, there are different frameworks available. For the .NET projects you will use the csUnit and NUnit frameworks. Projects in python use the PyUnit and DocTest frameworks. Java uses Junit or TestNG. There are also a few others, but this are the main contenders you will see in practice.
