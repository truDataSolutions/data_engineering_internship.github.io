Data Engineering Tools

Data engineers use multiple tools. In this section, we will touch upon the most common
tools used and build an end-to-end pipeline.

Azure

Azure is Microsoft's cloud computing suite of tools. It is commonly used for hosting databases in the cloud, but it also offers many other tools such as DevOps for project management, containers, migration, and so much more.

Azure Data Factory (ADF)
ADF is a cloud ETL/ELT tool that offers a code free environment to build data pipelines. It also supports multiple different database platforms such as Hadoop, Salesforce, Oracle, and many more.

Azure Blob Storage
Blob storage is an object storage for the cloud that is optimized for storing large amounts of unstructured data. This storage option is excellent for implementing data lakes as it is very scalable and cost effective.

Linked Service
Linked services is a connection between an internal and external resource. This is a key value pair that allows your pipeline to connect to a dataset and map it to where it needs to be placed.

Pipeline
Pipeline is where you define the transformations being performed on the data. You can copy data, create a look-up, conditional activities, and many others.


Amazon Web Services (AWS)

AWS is Amazons cloud computing suite of tools. One highlight of this service compared to others is the pay-as-you-go basis it is built on. Using AWS, you can build ETL/ELT pipelines in a code environment.

Elastic Computing (EC2)
EC2 is Amazons offering for renting virtual computers on which companies can run the AWS suite of tools on. On EC2, all the computer hardware is virtual meaning it can be run from anywhere.

Lambda
Lambda is an event-driven computing platform. It runs code in response to events and automatically manages the computing resources required by the code. This can be used to ingest code as well as transform it.

Simple Storage Solution (S3)
An S3 bucket is a cloud object storage that stores objects within a bucket. Buckets are extremely scalable and inexpensive making it a good choice to store all types of data.

Amazon Relational Database Service
ARDS is a relational database running in the cloud. If offers simplistic set-up, operation, and scaling for use with other applications. You can query data with many different querying languages including but not limited to Amazon Aurora, MySQL, and Microsoft SQL Server.

DynamoDB
Dynamo DB is a proprietary NoSQL database service that supports key-value pairs and document data structures.

Glue
Glue is an event-driven computing platform that runs code in response to events. It is used in place of Lambda if you have very large amounts of data being processed.

Redshift
Redshift is a data warehouse product. It is optimized for handling large scale data sets and data migrations. It also supports analytics making it quick and easy to produce analytics at scale.


Google Cloud Platform (GCP)

GCP is Googles cloud offering for data tools. It is not as popular as AWS or Azure yet but has shown tremendous growth over the last few years. Just like with the other two competitors, you can build ETL/ELT pipelines using Googles web applications. In GCP you can build pipelines on the command line using google cloud shell or code-free.

Dataflow
Dataflow is a service for executing several different data processing patterns. It is a pipeline tool that allows for both batch and streaming data. Dataflow is a ETL tool. For ELT pipelines you will need to use a different GCP tool.

BigQuery
BigQuery is an enterprise data warehouse that is built for machine learning, geospatial analysis, and business intelligence. It is capable of both ETL and ELT.

Storage
Storage is a service for storing you objects in Google Cloud. It is very similar to Amazon S3 as objects are stored in buckets.

Composer
Composer allows users to author, schedule, and monitor pipelines. It is built on Apache Airflow and is optimized for batch workflows. It allows for a code interface in python to manage your pipelines.
