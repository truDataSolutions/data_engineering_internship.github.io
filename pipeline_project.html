Pipeline Project

Using all the skills you've learned, you will build an proof of concept pipeline using the tools of your choice. The goal is to find (3) tables in saleforce that are populates, create a pipeline that will read these tables, do any necessary transformation and mappings, and load them into a new storage solution.

To help you visualize the concept behind this project here is a diagram (pipeline done in Azure).


From this diagram, you can see which tables have been used in the past. If you would like you can use these tables. Otherwise, search through and find tables that would interest you (must be populated). You will most likely need to create a mapping and look-up table. These tables are used to correctly connect the salesforce table and its contents to the storage table you created.

To add challenge to this project, make sure your inputs are parametrized so that they can take in any table and correctly output the data. This helps in creating a more robust pipeline.

Once you have finished building your pipeline, be sure to graph a high level chart to demonstrate the steps it goes through. Remember, the idea is to explain this to individuals who are not data engineers!
